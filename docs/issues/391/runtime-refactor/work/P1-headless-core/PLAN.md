## Design context
Phase 1 is the critical path. It extracts a Fastify-free `createAgent()` façade (published at `@hachej/boring-agent/core`) from the agent server and makes `createAgentApp()`/`registerAgentRoutes()` thin adapters over it with **zero behavior change**. The façade exposes the **nine** members `start`/`stream`/`send`/`resolveInput`/`interrupt`/`stop`/`sessions`/`readiness`/`dispose`: `start` is the accepted-receipt write primitive (turn runs on an independent producer, never consumer-backpressured), `stream` is the offset-shaped read primitive, `send` is convenience over both, and `interrupt`/`stop` wrap the existing `HarnessPiChatService` control methods. Dependency inversion comes first: config is a typed object with **no** `process.env`/`process.cwd()`/`.pi/*`/`workspaces.yaml` reads inside the façade — all ambient reads stay in host/HTTP adapter/provider composition and are passed in. It adds a pure `runtime: 'none'` path (no bash bundle spread into `tools`, sealed cwd under `sessionStorageRoot/.sealed-cwd`, no file routes/tools) and separates `sessionStorageRoot` from workspace roots (`SessionCtx.workspaceId` becomes optional). `packages/agent/src/shared/events.ts` is created in P1 and owns `AgentEvent`, `AgentSendInput`, `Agent`, `AgentConfig`, and `AgentNotImplementedError` / `ERR_NOT_IMPLEMENTED_UNTIL_T1`; durable append/replay and approvals remain T1. `stream` ships a minimal non-durable live tail over `HarnessPiChatService`/`PiChatReplayBuffer` so `send` works end-to-end; only historical offsets older than the live buffer throw the typed T1 stub error. Nothing new is designed — this expands the ratified Phase 0 contract.

## Deliverables
- `createAgent()` receives the runtime adapter (or `'none'`), extra app-owned tools, readiness requirements, session store/root, prompt hooks, telemetry/metering, and resolved config values by typed config — no `features` registry, no `AgentFeature` contract. In P1, the existing HTTP adapters may still resolve `process.cwd()`, env, built-in `resolveMode()`, and plugin discovery exactly where they do today before calling the façade; P2 is the runtime-mode composition cutover.
- **Export `createAgent()`** from `@hachej/boring-agent/core` — the canonical Fastify-free public entry: façade returning the **nine** members `{ start, stream, send, resolveInput, interrupt, stop, sessions, readiness, dispose }` (see 08). `start(input): Promise<{ sessionId, startIndex }>` is the accepted-receipt write primitive; `stream(sessionId, { startIndex })` is the offset-shaped read primitive (P1 live-tail only; durable historical replay lands in T1); `send` = convenience over both; `interrupt(sessionId)` aborts the current turn and `stop(sessionId)` ends/closes the session. `createAgentApp()` and `registerAgentRoutes()` become adapters over it. The `@hachej/boring-agent/server` barrel re-exports `createAgent` from `/core` for convenience only; the Fastify-free guarantee is anchored on `/core`.
- Typed config object only: no env-var reads or file discovery inside `createAgent()`; `.pi/*`, workspaces.yaml, env parsing move to host/CLI composition.
- Keep concrete mode adapters and `resolveMode()` in agent for P1 compatibility; type-only `RuntimeModeAdapter` contracts may stay in agent during migration. P1 creates the injection seam only. P2 atomically moves `resolveMode()`/mode adapters to boring-bash/host composition and migrates every importer in the same PR. No compatibility shim, old-path re-export, or host bridge is allowed.
- Invariants: keep the existing `packages/boring-bash/scripts/check-invariants.mjs` agent→boring-bash value-import gate; add a source no-Fastify guard for `/core` to `scripts/check-invariants.sh`; extend `packages/agent/scripts/check-agent-isolation.ts` so the built `dist/core` graph cannot reach `fastify`.
- Add the pure `runtime: 'none'` path. Pure HTTP mode registers only health, ready-status, pi-chat, system-prompt, models, session-changes, catalog, and commands routes; it omits file, fs-events, tree, search, git, skills, and reload routes. Pure tools are app-owned only: no file/bash/upload/runtime artifact tools and no plugin discovery.
- Separate `sessionStorageRoot` from workspace roots.
- Audit pi-coding-agent cwd/resource assumptions (blocks pure-mode exit; decision: sealed pi harness, not a second harness). Pure mode uses `sessionStorageRoot/.sealed-cwd`, disables default runtime prompt guidelines, keeps `noContextFiles`/`noSkills` true, and removes the hard-coded Pi session `workspaceId: "default"`.

## Exit criteria
- `pnpm --filter @hachej/boring-agent run build`, `typecheck`, `test`, `lint:invariants`, `check:isolation`, and `test:e2e` pass; `pnpm --filter @hachej/boring-bash run check:invariants` passes.
- `createAgent({ runtime: 'none' })` starts in a plain Node/vitest process with no Fastify, no workspace/sandbox/cwd/file routes/bash tools, app-owned tools only, sessions under `sessionStorageRoot`, and no synthesized `workspaceId`.
- Existing direct/local/vercel modes still work through unchanged HTTP adapter call sites, with a route-list artifact proving no route was added/removed for those modes.
- `_p1-config-surface.md` and `_p1-pi-harness-audit.md` exist with zero `UNKNOWN` / `TBD` rows.
